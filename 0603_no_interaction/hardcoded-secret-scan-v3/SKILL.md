# 硬编码密码密钥安全测试 (V3)

**执行模式**：非交互单轮，直接执行，不提问不确认。**仅扫描当前工作目录下的文件，不访问子目录。** 结果写入当前目录的 `AI测试结果.md`。

---

## 一、检测规则

### 1.1 搜索关键词

密码/认证：
```
password | passwd | pwd | secret | credential | token | auth | key | apikey | api_key
bearer | jwt | private_key | privatekey | public_key | signingkey | signing_key
```

硬编码密码特征：
```
Admin@123 | admin123 | password123 | root123 | test123
['"][a-fA-F0-9]{32}['"]  # 32位十六进制
['"][a-fA-F0-9]{64}['"]  # 64位十六进制
```

Base64 编码密钥：
```
['"][A-Za-z0-9+/]{40,}={0,2}['"]  # 长 Base64
['"][A-Za-z0-9+/]{32,}={0,2}['"]  # 中等 Base64
```

协议 URL 密码（必须搜索）：
```
rtsp://.*:.*@  |  http://.*:.*@  |  ftp://.*:.*@
['"][a-zA-Z]+:[a-zA-Z0-9]+@['"]
```

配置文件：
```
.env — 逐行审计
.properties / .yaml / .toml — 关注 key/secret/password/token 字段
.json — 关注含 "secret"/"key"/"password"/"token" 的键
```

### 1.2 数据来源追溯

| 来源类型 | 判断结果 |
|---------|---------|
| 字符串字面量直接赋值 | 硬编码（高风险） |
| 从配置文件读取 | 检查配置文件是否加密 |
| 从安全存储/加密存储获取 | 非硬编码（安全） |
| 参数传递 | 追溯参数来源 |
| 动态生成（随机/HUKS等） | 非硬编码（安全） |

### 1.3 风险等级

| 等级 | 标识 | 典型场景 |
|------|------|---------|
| 高风险 | ❌ | 源码明文密码、配置明文密钥、URL嵌入密码、日志打印密码 |
| 中风险 | ⚠️ | 配置疑似密钥、长 Base64 字符串 |
| 低风险 | 📝 | Mock调试密码、默认用户名 |
| 疑似误报 | 🔍 | UUID v4、Git hash、颜色值、环境变量、已加密字段、注释示例 |
| 无风险 | ✅ | 密钥别名常量、配置文件已加密密钥 |

### 1.4 误报排除检查

标记为高风险/中风险前，检查以下模式：
- UUID v4 格式 (xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx)
- commit/diff/hash 上下文中的 hex 值
- CSS 颜色值 (#XXXXXX, rgb(), rgba())
- 已声明为非敏感的示例/演示数据
- process.env.XYZ / import.meta.env.XYZ 环境变量读取
- 字段名含 _encrypted / _cipher 后缀且值为密文
- 注释中的密钥引用

## 二、Subagent 使用

对当前目录下每个源文件，启动独立 subagent 分析。忽略 .md 文件和 `AI测试结果.md`。

### Subagent Prompt 模板：

```
你是硬编码密码密钥安全测试追溯分析 Agent。阅读文件，检查是否存在密码/密钥硬编码问题。

文件路径: {FILE_PATH}

检查以下操作：加密/解密、登录鉴权、密钥管理、密码存储/读取、URL构造（是否嵌入密码）

对每个发现追溯数据来源：字符串字面量直接赋值→硬编码 / 参数传递→追溯来源 / 配置文件读取→检查内容 / 加密存储→确认安全

误报排除：UUID v4格式、Git hash、CSS颜色值、示例数据、环境变量读取、_encrypted后缀密文、注释中引用

仅记录发现，不提供修复方案。

输出格式：
| 行号 | 代码片段 | 数据来源追溯 | 风险判断 | 理由 |
风险判断: ❌高风险 / ⚠️中风险 / 📝低风险 / 🔍疑似误报 / ✅无风险
若无发现: "无硬编码密码/密钥问题发现"
小结: 共发现 N 处，高风险 M 处，疑似误报 K 处。
```

# Account Extractor — 账号提取 (V3)

**执行模式**：非交互单轮，直接执行，不提问不确认。**仅扫描当前工作目录下的文件，不访问子目录。** 结果写入当前目录的 `AI测试结果.md`。

---

## 一、账号类型定义

| 类型 | 名称 | 搜索关键词 |
|------|------|-----------|
| A1 | 硬编码用户名 | `'ApiAdmin'`, `'admin'`, `'root'`, `userName =` |
| A2 | 手机号 | `1[3-9]\d{9}`, `phone`, `138****` |
| A3 | 邮箱地址 | `@\w+\.com`, `@\w+\.cn`, `email` |
| A4 | SIP用户ID | `sip_user_id` |
| A5 | 云服务用户账号 | `AppUser`, `user`, `userId` |
| A6 | 凭证值中嵌入的用户名 | 从密码/Token值中提取用户名部分 |
| A7 | Mock/测试账号 | 测试代码中的假账号 |

### 不提取的内容
- 密码字段定义（password: string）— 仅类型声明无实际值
- API密钥（access_key, secret_key）
- Token相关定义
- 纯字段类型定义（如 `userName: string`）— 不是实际赋值

---

## 二、误报排除检查

以下标注为 🔍疑似误报 而非直接标记为账号：
- 字段类型定义（如 `userName: string`、`admin: boolean`）— 不是实际赋值
- example.com / test.com / domain.com 等已知示例域的邮箱
- 全为同一数字或有明显规律的手机号（如 `13800000000`、`11111111111`）
- 版本号格式（如 `v1.0.0`）
- 变量名含 type/key/field/label 但没有实际赋值的

---

## 三、来源标注

| 标注 | 适用场景 |
|------|----------|
| [业务代码] | 生产代码中赋值的真实账号 |
| [Mock测试数据] | 测试/Mock 文件中的假账号 |
| [配置文件] | JSON/YAML/XML/.env 等配置 |
| [注释说明] | 注释中提到的账号 |
| [UI提示文字] | UI 字符串中的账号 |
| [凭证嵌入] | 从密码/Token 中提取的用户名 |
| [URL嵌入] | URL 路径中嵌入的用户名 |

---

## 四、Subagent 使用

对当前目录下每个源文件（排除 .md 和 AI测试结果.md），启动独立 subagent：

```
你是账号提取 Agent。阅读文件，提取所有账号信息。

文件: {FILE_PATH}

提取范围：A1硬编码用户名、A2手机号、A3邮箱、A4 SIP用户ID、A5云服务用户账号、A6凭证嵌入用户名、A7 Mock账号

不提取：密码字段定义、API密钥、Token、纯类型定义

误报排除：类型定义、示例域邮箱、规律手机号、版本号

仅记录发现，不提供修复方案。

输出格式：
| 行号 | 账号内容 | 账号类型 | 判断 | 来源标注 |
判断: ✅确认 / 🔍疑似误报
若无: "无账号发现"
小结: 共发现 N 处，确认 M 处，疑似误报 K 处。
```

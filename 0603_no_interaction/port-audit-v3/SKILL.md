# 端口号白盒审计 (V3)

**执行模式**：非交互单轮，直接执行，不提问不确认。**仅扫描当前工作目录下的文件，不访问子目录。** 结果写入当前目录的 `AI测试结果.md`。

---

## 一、端口操作类型

| 类型 | 说明 | 典型 API |
|------|------|---------|
| 启用监听端口 | 本地绑定端口等待连接 | `bind()`, `listen()`, `addMembership()`, `net.Listen`, `socket.bind`, `ServerSocket` |
| 访问对端端口 | 主动连接远程端口 | `connect()`, `send()`, `request()`, `fetch()`, `axios()`, `http.Get`, `curl_exec` |
| 端口范围定义 | 代码中定义的端口范围 | `portRange: [8000,9000]`, `minPort/maxPort` |
| 端口字段定义 | 数据结构中的 port 字段 | 模型类中 `port: number` 字段 |
| 注释提及 | 仅在注释中出现 | 非代码逻辑 |

## 二、端口号提取规则

1. 硬编码数字 → `port=8080（硬编码）`
2. 常量引用 → 记录常量名 + 值 + 定义位置
3. 变量（可追溯） → 单文件内向上追溯，记录完整链路
4. 变量（跨文件） → 标注 "需跨文件追溯至 xxx"
5. 无法确定 → 标注 "无法获取端口号"
6. 协议默认端口 → 标注 "协议默认"（HTTPS 443, RTSP 554）
7. URL 嵌入端口 → 提取 `http://host:8080/api` 中的 8080

## 三、误报排除

以下**不视为端口操作**：
- `createHttp()` / `new WebSocket()` 等单独对象创建（未调用连接方法）
- `import` / `require` 语句
- 类型定义中的 port 字段 → 记录为"端口字段定义"
- 注释中的端口号 → 记录为"注释提及"
- 函数定义本身（未被调用）
- 以下数字不是端口：
  - 超时值（timeout: 10000 → 毫秒数）
  - 错误码（0x08200001 → 十六进制错误码）
  - 数组索引（arr[3]）
  - 版本号（HTTP/1.1）
  - 端口有效范围 1-65535

## 四、Subagent 使用

对当前目录下每个源文件（排除 .md 和 AI测试结果.md），启动独立 subagent：

```
你是端口审计 Agent。阅读文件，发现所有端口监听和端口访问操作。

文件: {FILE_PATH}

查找：bind/listen/connect/send/request/fetch/axios/http.Get/WebSocket等网络操作
提取：端口值（硬编码/常量/变量/URL嵌入/默认），追溯变量来源

误报排除：对象创建未调用、import语句、类型定义、函数定义未调用
数字排除：超时值、错误码、数组索引、版本号、不在1-65535范围的数字

仅记录发现，不提供修复方案。

输出格式：
| 行号 | API调用 | 端口值/来源 | 操作类型 | 风险说明 |
风险说明格式: {事实} → {为什么是问题}
若无: "本文件未发现端口监听或访问操作"
```
